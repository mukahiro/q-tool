# 要件定義書

本プロジェクトで開発する「学生と教師をつなぐリアルタイム質問ツール」の目的、機能、および非機能要件を定義します。

## 1. プロジェクト概要
- **アプリケーション名:** (未定)
- **目的:** 授業中の学生からの質問をリアルタイムで収集し、AI要約によって教師が効率的に回答フィードバックを行える環境を提供する。
- **ターゲットユーザー:** 教師（ルーム作成・管理）、学生（質問投稿・閲覧）

## 2. ユーザー機能要件

### 2.1 認証・ユーザー管理
- **教師:** Supabase Authを利用したサインアップ、ログイン、ログアウト。
- **学生:** 原則として認証なし（または簡易的なニックネーム入力）での入室を想定（参加のハードルを下げるため）。

### 2.2 ルーム管理（教師用）
- ルームの新規作成。
- 入室用QRコードおよびURLの生成・表示。
- ルームの開始・終了管理。

### 2.3 質問投稿・閲覧（学生用）
- QRコード/URLからの簡易入室。
- 授業中のリアルタイムな質問投稿。
- 他の学生の質問に対する「いいね（リアクション）」機能（優先度の可視化）。

### 2.4 セクション管理・AI要約（教師用）
- 「セクション終了」ボタンの操作。
- **授業への集中を優先するため、ボタンを押すまでは個別の質問内容は教師画面に表示されない。**
- ボタン押下をトリガーとして、該当セクション内に投稿された質問群をAIで自動要約・カテゴリ分けして表示。
- 要約された内容に基づいた、教師による口頭回答のサポート。

### 2.5 拡張機能（将来的な追加検討）
- **講義資料の表示:** PDFやスライド資料をアプリ内で閲覧できる機能。
- **質問への返信・議論:** 投稿された質問に対して、教師や他の学生が返信（スレッド形式）できる機能。
- **授業後のアーカイブ:** 質問とAI要約、および行われた議論のログを保存・閲覧できる機能。

## 3. 非機能要件
- **リアルタイム性:** 学生側の質問投稿やリアクションの反映（学生同士で閲覧する場合）。教師側はステータス管理（件数のみの表示など）に限定。
- **AI連携:** Google AI Studio (Gemini API) を利用した質問内容の要約機能（セクション終了時のバッチ処理）。
- **パフォーマンス:** 多くの学生が同時接続しても遅延の少ない設計。
- **レスポンシブ対応:** 学生はスマートフォン、教師はPC/タブレットでの利用を主眼に置いたUI/UX。

## 4. UI/UX方針
- 授業を妨げない、シンプルかつクリーンなデザイン。
- 学生側：直感的な投稿インターフェース。
- 教師側：一目で質問状況を把握できるダッシュボード。

## 5. 今後の検討事項
- 授業後のアーカイブ機能（質問と回答要約の保存）。
- 不適切な投稿のフィルタリング機能。
- 複数セクション（章立て）の管理。
