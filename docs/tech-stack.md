# 技術スタックの検討と決定

本プロジェクトで採用する技術スタックとその選定理由をまとめます。

## フロントエンド・フレームワーク

### **Next.js (App Router)**
- **選定理由:** Reactベースの最新フレームワークであり、サーバーコンポーネント (RSC) による高いパフォーマンスと、Vercelへのデプロイの親和性を考慮しました。
- **ルーティング:** ファイルベースのApp Routerを使用し、モダンな開発エクスペリエンスを優先します。

## 開発言語

### **TypeScript**
- **選定理由:** 静的型付けによるバグの早期発見、コードの自己文書化、および大規模開発における保守性の向上を目的としています。

## スタイリング

### **Tailwind CSS**
- **選定理由:** ユーティリティファーストな設計により、スタイル定義の高速化とデザインの一貫性を保ちます。Next.jsとの相性も良く、標準的な構成として採用しました。

## データベース

### **Supabase (PostgreSQL)**
- **選定理由:** 
  - マネージドなPostgreSQL環境を迅速に構築可能。
  - 将来的な認証 (Auth)、リアルタイム機能、ストレージ機能との統合が容易。
  - スケーラビリティと、開発初期の導入コストの低さを評価しました。

## 認証

### **Supabase Auth**
- **選定理由:** 認証基盤の構築・管理をSupabaseに完全に委ねることで、セキュリティリスクの低減と開発工数の削減を図ります。Supabase Databaseとの親和性が非常に高く、行レベルセキュリティ (RLS) との連携もスムーズであるため採用しました。

## ORM (Object-Relational Mapping)

### **Prisma**
- **選定理由:** 
  - 強力な型生成機能により、TypeScriptとの親和性が非常に高い。
  - スキーマ定義が直感的で、マイグレーション管理が容易。
  - 直感的なAPIによる開発効率の向上。

## AI API

### **Google AI Studio (Gemini API)**
- **選定理由:** 高性能な言語モデル Gemini を利用でき、開発者向けの無料枠や使いやすさを考慮して採用しました。質問の要約やカテゴリ分けに使用します。

## インフラ・デプロイ

### **Vercel**
- **選定理由:** Next.jsの開発元であり、ゼロ構成でデプロイ・プレビュー環境の構築が可能。サーバーレス環境との統合も最適化されています。
